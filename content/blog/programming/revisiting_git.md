I have been using Git for years, mostly as a practical tool to manage my code and keep track of changes.

However, I recently realized that my understanding is often superficial. I know how to use the main commands, but I don’t always fully understand what is happening behind them.

For this reason, I want to go back to Git with a different approach: not just using it, but trying to understand its logic more deeply.

This post is just a starting point. In the next ones, I will try to clarify key concepts and improve my workflow step by step.
